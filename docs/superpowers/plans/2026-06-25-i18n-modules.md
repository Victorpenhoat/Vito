# Traduction des modules EN/IT/ES Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Traduire les 11 namespaces modules (262 clés) en EN, IT, ES, sous garde-fou de parité (clés + placeholders ICU identiques à fr).

**Architecture:** Un test de parité protège la structure ; chaque langue est traduite dans un fichier `messages/<loc>.json` (valeurs seulement, clés/placeholders inchangés). FR = source.

**Tech Stack:** next-intl, Vitest, Playwright.

## Global Constraints

- Ne changer **que les valeurs** des namespaces modules (`vins, gouts, recherche, restos, voyages,
  depenses, abonnement, conciergerie, famille, agence, admin`). **Jamais** une clé, ni un placeholder ICU
  (`{n}`, `{x}`…), ni les namespaces déjà traduits (`app, auth, nav, shell, accueil`).
- FR (`messages/fr.json`) = source de vérité. JSON valide.
- Registre **informel** (IT `tu`, ES `tú`). Terminologie cohérente par namespace.
- Gate par task de traduction : le **test de parité** vert + `npm run build` (JSON valides) + `npm run lint`.

### Task 1: Test de parité i18n (garde-fou)

**Files:**
- Create: `src/lib/i18n/messages-parity.test.ts`

- [ ] **Step 1: Écrire le test**

Create `src/lib/i18n/messages-parity.test.ts` :
```ts
import { describe, it, expect } from "vitest";
import fr from "../../../messages/fr.json";
import en from "../../../messages/en.json";
import itMsg from "../../../messages/it.json";
import es from "../../../messages/es.json";

type Json = { [k: string]: unknown };

function leaves(obj: Json, prefix = "", acc: Record<string, string> = {}): Record<string, string> {
  for (const [k, v] of Object.entries(obj)) {
    const path = prefix ? `${prefix}.${k}` : k;
    if (v && typeof v === "object") leaves(v as Json, path, acc);
    else acc[path] = String(v);
  }
  return acc;
}
function placeholders(s: string): string[] {
  return [...s.matchAll(/\{(\w+)\}/g)].map((m) => m[1]).sort();
}

const frLeaves = leaves(fr as Json);
const frKeys = Object.keys(frLeaves).sort();

describe.each([
  ["en", en],
  ["it", itMsg],
  ["es", es],
])("parité i18n %s", (name, locale) => {
  const locLeaves = leaves(locale as Json);
  it("a exactement le jeu de clés de fr", () => {
    expect(Object.keys(locLeaves).sort()).toEqual(frKeys);
  });
  it("préserve les placeholders ICU de chaque clé", () => {
    for (const k of frKeys) {
      expect(placeholders(locLeaves[k] ?? "")).toEqual(placeholders(frLeaves[k]!));
    }
  });
});
```
(Import JSON via chemin relatif depuis `src/lib/i18n/`. `resolveJsonModule` est actif — sinon ajuster
l'import. Le test passe immédiatement : en/it/es sont déjà structurellement identiques à fr.)

- [ ] **Step 2: Lancer**

Run: `npx vitest run src/lib/i18n/messages-parity.test.ts`
Expected: PASS (6 assertions — 2 par locale). Si import JSON KO, vérifier `resolveJsonModule` dans
tsconfig (probablement déjà activé) ou importer via `assert { type: "json" }`/lecture fs.

- [ ] **Step 3: Commit**

```bash
git add src/lib/i18n/messages-parity.test.ts
git commit -m "test(i18n): parité clés + placeholders fr/en/it/es (garde-fou traduction)"
```
